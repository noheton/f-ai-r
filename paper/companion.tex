\begin{abstract}
This is the companion technical report to the \faiar\ position paper
\todo{insert main-paper DOI once \texttt{paper/publication-consent.md}
is signed off}. The position paper argues that FAIR is insufficient
for LLM-assisted scholarly writing and proposes a multiplicative
extension --- F(AI)\textsuperscript{2}R --- enforced by eight
integrated practices and a ten-stage agent pipeline; this companion
carries the artefacts the argument leans on. Seven sections cover, in
order, the verification-ladder specification used by the Source
Analyzer and Aligner stages, the prompt taxonomy under
\texttt{agents/}, the F(AI)\textsuperscript{2}R cell-mapping
crosswalk (FAIR sub-principles \(\times\) AI passes), the per-step
process-evolution chronology, the four diagnostic SPARQL queries on
the live \texttt{doc/provenance.ttl} graph with worked numbers, the
repository implementation (layout, CI, replay), and two forward-looking
pattern extensions (domain ontologies and graph interconnectivity).
The seven sections are referenced from the main paper via
\texttt{\textbackslash companionref}; their \texttt{fair2r:Claim}
IRIs continue to resolve in the same shared
\texttt{doc/provenance.ttl} graph, which spans both documents.
\end{abstract}
