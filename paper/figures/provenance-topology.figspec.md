## Caption (one sentence)
A node-link preview of the F(AI)\textsuperscript{2}R provenance
graph: five PROV-O lanes (Agents, Plans, Activities, Entities,
Claims) with a satellite band of Contribution mirrors, each node
decorated with its instance count from the live
`doc/provenance.ttl`.

## Why matplotlib (not graphviz, not TikZ)
A graphviz auto-layout would let the data drive node placement,
but the figure's job is the opposite --- to show that the *schema*
has the shape the methodology describes (five lanes, left to
right, plus a downward Contribution mirror). Hand-laid columns
are honest about that intent; a Python script keeps the counts
faithful to the graph because they are read at render time.
TikZ would re-encode the counts by hand and drift on the next
provenance audit. matplotlib + `FancyArrowPatch` is the cheapest
honest medium.

## Data / relations the figure must show
- Five lane columns (Agents, Plans, Activities, Entities, Claims)
  with a hairline rule under each header.
- Schema nodes per lane, sized identically so cardinality reads
  from the badge rather than the box.
- A `n = <count>` badge on every node, populated by `rdflib`
  parsing `doc/provenance.ttl` at render time.
- Backbone arrows for the PROV-O wiring (Agent
  `wasAssociatedWith` Activity; Activity `used` / `generated`
  Entity; Source `wasDerivedFrom` Claim; Claim `appearsIn`
  Manuscript). Only one representative label per backbone arc to
  avoid clutter.
- A satellite Contribution band underneath, dotted edges
  carrying the `contributedBy` mirror that links Agents to
  HumanContribution / AIContribution / MetaContribution.

## Style
DLR Corporate Design palette: `dlrBlau1` `#00658B` (Claim, the
load-bearing class), `dlrBlau5` `#D1E8FA` (AIAgent, Prompt),
`dlrGruen5` `#E6EAAF` (HumanResearcher, HumanContribution),
`dlrGelb5` `#FFF8BE` (MetaContribution), `dlrGrau5` `#EBEBEB`
(Entities lane), `dlrGrau1` `#4D5258` (text, hairlines), `dlrGrau3`
`#B1B1B1` (header rules). Hairline 0.4\,pt strokes throughout;
square corners; sans-serif (Helvetica / Arial fallback DejaVu
Sans). The AI-only nodes carry a `//` hatch in addition to the
soft-blue fill so the figure remains legible in greyscale.

## Placement
- Loaded by `paper/sections/provenance-analysis.tex` immediately
  before the rung-distribution table and figure, so the reader
  sees the *shape* of the graph before reading the audit numbers
  drawn from it. Cross-referenced as
  `Figure~\ref{fig:provenance-topology}`.
- README / Pages topology preview can pick up the PNG; it is the
  only figure in the set that doubles as a graph-shape preview.
