## Caption (one sentence)
A single-glance companion to the rung-distribution table:
horizontal stacked bars showing how many \texttt{fair2r:Source} and
\texttt{fair2r:Claim} entities sit at each verification rung at
submission time.

## Why matplotlib (not TikZ)
The figure is data-bound: counts come from the same generator that
fills the auto-generated table at
`paper/sections/_generated/rung-distribution.tex`. Driving the
figure from the same data path keeps it in lock-step with the
table; a Python script with a deterministic palette is the cheapest
honest way to do that. TikZ would re-encode the data by hand and
drift on the next regeneration.

## Data / relations the figure must show
- Two stacked bars, one per entity class (`fair2r:Source` row
  on top, `fair2r:Claim` row on bottom).
- One segment per rung in reading order:
  \textsc{needs-research}, \textsc{lit-retrieved},
  \textsc{ai-confirmed}, \textsc{lit-read} /
  \textsc{human-confirmed}, \textsc{source-vendored}.
- Numeric label centred on each segment that has visual room
  (white text on the dark `dlrBlau1` segment, mid-grey
  `#4D5258` text otherwise).
- Right-edge total per row with the entity-class label.
- A bottom-row legend listing only the rungs that occur in the
  data.
